The Development of User Interface Design

What does UI mean?

  • A user interface (UI) is about the point of interaction between the user and a digital device or product

Who invented UI?

  • Alan Kay, Douglas Engelbart, and other researchers at Xerox PARC

When was the first UI made?

  • 1973

Why is UI design important?

  • competitive advantage, brand loyalty, enhanced engagement, customer acquisition, and customer retention.

The User Interface of a website may be considered one of the more important aspects of designing a website. The User Interface allows the user to interact with the website they are viewing. The User Interface also deals with a large amount of the visual aspects of a website. The UI is about functionality and seeing what works well together. It helps to know coding for UI design but it is not fully necessary. The first UI was called the graphical user interface (GUI). The GUI was not intended for personal use and was used mostly for research in universities. In 1983, Apple introduced the Lisa, the first computer with a GUI. The development of the UI that we know today has taken 50+ years to get to. Progressing the design of the UI has become more prevalent in recent years.

Milestones

    Xerox Star Workstation

    • The Xerox Star Workstation was the first commercial personal computer. It was officially named the Xerox 8010 Information System. It had a bitmapped display and window-based GUI. It functioned with icons, folders, mouse (two-button), Ethernet networking, file servers, print servers, and e-mail.The main idea of the user interface was to imitate the office paradigm in order to make it intuitive for users. The Xerox Star user interface was based on the concept of objects. For example, a word processing document would hold page objects, paragraph objects, sentence objects, word objects, and character objects. The user could interact with objects by clicking on them with the mouse, and by pressing dedicated keys on the keyboard to cause standard object functions (open, delete, copy, move). There was also a "Show Properties" key used to display settings, called property sheets, for the particular object (e.g., font size for a character object). These general set-ups simplified the structure of all the programs.

    Skeuomorphism in UI

    • Skeumorphism is a digital imitation of an object that exists physically. The shape, texture, and functionality of the object is imitated for design and sometimes for function.It was invented by Dr. Henry Colley March in 1889. Skeuomorphism was first used in UI design in the 80's and since has fallen in and out of popularity amongst UI and UX designers. It was once very popular with Steve Jobs and could be seen all over Apple products. The concept of Skeuomorphism was meant to improve user interaction and experience but with the rise of other ideas to aid this it isnt very commonly seen anymore. Another factor in why it isnt used anymore is the fact that high resolution bitmaps perform poorly on smartphones. More recently some UI and UX designers have been making arguments for Skeuomorphism and believe it can still serve a good purpose in designing things for the digital world to enjoy.

    Human Interface Guidelines

    • Human Interface Guidelines are a type of development document. They are used to help UI designers when developing something like an app or a website. Human Interface Guidelines improve user experience by incorporating consistency and being intuitive. The guidelines are typically used to enstate a common look and feel across what is being developed and where, They also list policies. The policies are sometimes based of of human-computer interaction but are often just based on a developers preferences. The guidelines are intended to determine everything about a website or application but sometimes developers can break the guidelines if they feel that the given guidelines dont fully fit with the idea of what they are trying to develop or its functionality and usability But in turn the organization publishing the Human Interface Guidelines might withhold endorsement of the application.
